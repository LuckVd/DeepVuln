## 五条强制规则

| 规则 | 条件 | 错误类型 |
|------|------|----------|
| Rule 1 | exploitability 与 final_status 不匹配 | GlobalAdjudicationError |
| Rule 2 | 同 ID 存在 EXPLOITABLE + NOT_EXPLOITABLE | GlobalAdjudicationError |
| Rule 3 | 跨引擎状态冲突 | GlobalAdjudicationError |
| Rule 4 | 状态等级逆向升级 (gap >= 2) | GlobalAdjudicationError |
| Rule 5 | final_status 为空 | GlobalAdjudicationError |

## 设计原则

**本阶段是:**
- 全局一致性校验
- 冲突禁止机制
- 强一致裁决

**本阶段不是:**
- 修改评分逻辑
- 自动合并冲突
- 静默修正错误

## 预期效果

- 强一致性: 同一漏洞 ID 状态完全一致
- 冲突禁止: 不允许逻辑矛盾
- 确定性: deterministic 输出
- 可追溯: metadata 记录检查结果
- 显式错误: 冲突必须抛异常
