## Bug 修复详情

### 问题
当项目路径包含 `target`、`test` 等关键词时（如 `/opt/target/OWASP_Juice_Shop`），`_should_skip_path` 方法会错误地跳过所有源文件，导致攻击面检测返回 0 个入口点。

### 原因
`_should_skip_path` 检查的是绝对路径的所有部分，而不是相对于源代码目录的部分。

### 解决方案
- `_should_skip_path` 新增 `source_path` 参数
- 只检查相对路径部分，避免绝对路径中的关键词误判

### 效果
- OWASP Juice Shop: 0 入口点 → 418 入口点
- 扫描文件: 0 → 486
