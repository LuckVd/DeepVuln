## 技术细节

### P1: 漏洞模式库
- `VULNERABILITY_PATTERNS` 字典包含 SQL 注入、XSS、命令注入、路径遍历、SSRF、反序列化、代码注入
- 每种类型包含多语言的 vulnerable_patterns、sinks、sanitizers
- `get_vulnerability_patterns_prompt()` 生成动态 Prompt

### P2: 攻击者视角
- `ATTACKER_PERSPECTIVE` 常量引导 LLM 思考：
  - 如果攻击者控制输入会怎样？
  - 如何绕过防御？
  - 边缘情况（编码、null 字节、竞态条件）

### P3: 可疑代码机制
- JSON 输出新增 `suspicious_code` 数组
- `_convert_suspicious_to_finding()` 转换为 Finding 对象
- metadata 中标记 `is_suspicious=True`

### Bug 修复
- `_should_skip_file()` 只检查相对路径
- 修复 `/opt/target/` 路径被误跳过的问题

## 测试结果

- 优化后：50+ 确认漏洞，78 个可疑代码
